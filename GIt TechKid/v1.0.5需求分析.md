# TechKids V1.0.5 需求分析

| 项目   | 内容                                                                                                   |
| ---- | ---------------------------------------------------------------------------------------------------- |
| 分析版本 | V1.0.5                                                                                               |
| 输入材料 | `TechKids_PRD_V1.0.4.md`、`TechKids_Demo.html`、`GIt TechKid/海外市场App投资价值分析报告.md`、`GIt TechKid/部分要求.md` |
| 分析日期 | 2026-05-05                                                                                           |
| 目标   | 在 V1.0.4 的 AI 安全与创意工坊基础上，补齐交付闭环，并将产品方向对齐海外 EdTech / Kids Coding 竞品机会                                 |

## 一、当前版本判断

V1.0.4 已经把产品从「好看的单文件 Demo」推进到「具备生产化 AI 架构意识」：PRD 明确了后端代理、CORS、Safe Init、System Prompt、Few-Shot 等能力，Demo 中也已经把 AI 请求指向 `http://127.0.0.1:8765/chat`，并保留了安全兜底回复。

但当前仓库存在一个关键交付缺口：PRD 中提到的 `server.py`、`start_proxy.bat`、`README_PROXY.md` 尚未出现在文件系统中。也就是说，V1.0.4 的「后端代理模式」在文档和前端配置层面成立，但还没有形成可复现的 Python 运行包。用户提到的 Python 脚本在当前目录下未检索到，这是 V1.0.5 最优先修复项。

## 二、海外竞品与市场分析对 TechKids 的启发

你的海外市场报告将 EdTech 排在高价值赛道，核心趋势是游戏化学习、微学习、AI 私人导师、VR/AR 沉浸体验和创作者角色。其中「儿童编程教育」被标为高增长方向，适合 TechKids 当前的产品定位。

结合报告与海外竞品，TechKids 的 V1.0.5 不应该只继续堆页面，而应转向三个关键词：

1. **AI 私教化**：从「能聊天」升级为「能按年龄、关卡、错误原因给出下一步指导」。
2. **学习路径化**：从「几个演示关卡」升级为「可解释的概念进阶路径」。
3. **创作闭环化**：从「模板与详情页」升级为「创建、编辑、保存、提交挑战、家长查看成果」的完整链路。

| 竞品 / 方向 | 可借鉴点 | TechKids 应避开的同质化 |
|---|---|---|
| codeSpark | 低文字门槛、儿童自驱式闯关、从谜题过渡到创作 | 不只做低龄拼图，应保留 AI 与家长报告差异化 |
| Tynker / Tynker Copilot | 课程路径、AI 编程助手、从积木到真实语言的进阶 | 不直接复制课程商城，应先做 Demo 内可验证的 AI 指导闭环 |
| ScratchJr / Scratch | 故事、游戏、动画的儿童创作心智 | 不能只停留在作品展示，需要真正的编辑与保存 |
| Khan Academy Kids | 家长信任、免费、无广告心智、宽学科基础 | 当前 IAA 变现要更克制，广告必须家长可控 |
| Swift Playgrounds | 从学习到真实 App 开发的成长通道 | TechKids 暂不需要进入真实 Swift 开发，但可设计「积木到代码」预览 |
| Roblox / UGC 平台 | 创作者经济与作品社区有高增长空间 | 儿童开放社区风险高，V1.0.5 应先做家长可控分享 |

## 三、V1.0.5 版本主题

**建议主题：安全可交付的 AI 私教 + 学习与创作闭环。**

这个主题贴合 V1.0.4 已完成的 AI 安全架构，也贴合海外竞品分析中的 AI 私人导师、游戏化学习、创作者经济和隐私优先趋势。V1.0.5 不建议把重心放在新增大量视觉页面，而是优先让核心路径变得可运行、可验证、可持续迭代。

## 五、详细需求建议

### V105-P0-01 Python AI 代理交付包补齐

**背景**：PRD V1.0.4 已定义后端代理，但当前仓库没有 Python 脚本。Demo 中 `DS_CONFIG.endpoint` 已指向本地代理，导致真实 AI 体验依赖一个尚未交付的文件。

**建议功能**：

- 新增 `server.py`：提供 `/health` 与 `/chat`。
- 新增 `.env.example`：声明 `DEEPSEEK_API_KEY`、`PORT`、`MODEL`。
- 新增 `start_proxy.bat`：Windows 一键启动。
- 新增 `README_PROXY.md`：说明安装、启动、测试、常见错误。
- 前端增加代理状态提示：未连接代理时显示「当前为安全演示模式」。

**验收标准**：

- `python server.py` 可启动。
- `GET /health` 返回服务状态。
- `POST /chat` 能转发到 DeepSeek 并返回标准 `choices[0].message.content`。
- `TechKids_Demo.html` 全局搜索不到真实 API Key。
- 代理不可把完整儿童隐私输入写入日志。

### V105-P0-02 显式 Few-Shot 示例库与输出复检

**背景**：PRD 已说明 Few-Shot 是 V1.0.5 推荐增强点。当前实现主要通过 System Prompt 约束风格，还没有把高质量示例注入到 `messages` 中。

**建议功能**：

- 建立 `FEW_SHOT_EXAMPLES`：覆盖变量、循环、条件、坐标、调试、隐私拒答、跑题引导。
- 按问题类型动态选择 2 至 4 条示例，避免上下文过长。
- 增加输出复检：长度、是否询问隐私、是否偏离儿童编程、是否暴露模型身份。
- 增加「解释我的指令」能力：用户运行失败后，小智根据当前关卡、指令序列、目标坐标给出下一步提示。

**验收标准**：

- 10 个安全测试问题均被正确拦截或引导。
- 编程概念回答稳定保持 80 至 200 字。
- 回答中至少包含一个儿童可理解的类比或 TechKids 场景引用。
- 失败兜底时仍能返回儿童安全话术。

### V105-P0-03 学习路径与概念地图

**背景**：海外 Kids Coding 产品的竞争力通常不只是小游戏，而是清晰的概念进阶。当前 Demo 只有少量关卡配置，首页虽写「30+ 关卡」，但实际关卡数据尚未支撑。

**建议功能**：

- 将关卡扩展为 4 条概念线：顺序、循环、条件、变量。
- 每条概念线至少 5 个可演示关卡，总计 20 个 MVP 关卡。
- 增加「概念地图」：家长和儿童都能看到当前正在学什么。
- 增加失败原因提示：路径不够、方向错误、碰到障碍、未使用目标概念。
- 增加积木到伪代码预览，为后续真实代码学习铺路。

**验收标准**：

- 每个关卡配置包含 `concept`、`difficulty`、`goal`、`solution`、`hint`。
- 完成关卡后写入本地学习记录。
- 家长中心可按概念展示完成率。
- 首页模块文案与实际关卡数量一致。

### V105-P1-01 学习报告 2.0 与安全摘要

**背景**：Khan Academy Kids 等产品强依赖家长信任。TechKids 当前家长中心有数据卡片，但还不够解释「孩子到底学会了什么」。

**建议功能**：

- 周报卡片：学习时长、完成关卡、概念掌握、创作数量。
- AI 安全摘要：拦截次数、跑题引导次数、敏感请求类型，不展示完整聊天隐私。
- 广告设置：允许/禁止激励广告、每日上限、仅家长确认后展示。
- 使用时长设置：30/45/60 分钟档位。

**验收标准**：

- 家长中心能看到概念级进度，而不只是百分比。
- 关闭广告后，奖励按钮改为非广告替代任务。
- 安全摘要不展示儿童原始敏感输入。

### V105-P1-02 创意工坊编辑器 MVP 与挑战提交闭环

**背景**：当前创意工坊已有模板、草稿、详情页，但编辑动作仍是 Toast 提示。对标 ScratchJr / codeSpark，创作模块必须从「展示」进入「可编辑」。

**建议功能**：

- 轻量编辑器 MVP：角色、背景、动作积木、播放预览。
- 模板创建后进入编辑器，而不是只生成草稿卡片。
- 草稿保存版本号与最近编辑时间。
- 每周挑战增加「提交作品」按钮，提交后进入作品集。

**验收标准**：

- 用户能从模板进入编辑器，修改至少 1 个角色动作并保存。
- 刷新页面后草稿可恢复。
- 挑战进度不再只靠点击按钮增长，而是和作品提交关联。

### V105-P1-03 儿童安全分享与作品集

**背景**：海外报告强调创作者经济，但儿童产品不能直接开放社交。TechKids 适合先做「家长可控分享」。

**建议功能**：

- 我的作品集：已发布、草稿、挑战作品分类。
- 分享前家长确认。
- 生成本地模拟分享链接或二维码占位。
- 作品公开状态分为：仅自己、家庭可见、课堂码可见。
- 不开放评论区，先用预设鼓励反馈替代。

**验收标准**：

- 分享流程必须经过家长门槛。
- 作品不包含儿童真实个人信息字段。
- 排行榜与作品集不展示真实姓名。

### V105-P1-04 多语言与地区化基础

**背景**：本次已新增 README_CN / README_TW / README_EN，说明项目开始具备海外表达需求。下一步应把应用文案也从硬编码中抽离。

**建议功能**：

- 新建 `i18n` 文案字典：`zh-CN`、`zh-TW`、`en-US`。
- AI Prompt 增加 locale 参数，支持中文、繁中、英文回答风格。
- 家长中心增加地区化隐私提示。
- README 与 PRD 版本号同步更新。

**验收标准**：

- 首页、AI 实验室、创意工坊、家长中心核心文案可切换语言。
- AI 回答语言跟随界面语言。
- 不同语言下按钮文字不溢出。

### V105-P2-01 家长可控的 IAA / 去广告实验

**背景**：当前产品变现模式为 IAA，但海外儿童教育产品中，广告会明显影响家长信任。Khan Academy Kids 的免费无广告心智与 Tynker/codeSpark 的订阅心智都说明：儿童教育场景需要更谨慎的商业化表达。

**建议功能**：

- 激励广告每日次数上限。
- 家长中心提供「关闭广告」选项。
- 关闭广告后用学习任务替代双倍奖励。
- 增加「家庭通行证」Mock：无广告、更多关卡、更多创作模板。

**验收标准**：

- 儿童侧不能绕过家长设置开启广告。
- 广告奖励不成为完成学习的唯一通路。
- README 明确当前为商业化演示，不接入真实支付。

### V105-P2-02 学习行为埋点与本地分析

**背景**：海外报告提示 CAC、LTV、留存与数据飞轮风险。即使 Demo 阶段，也应开始定义事件模型。

**建议事件**：

- `level_start`、`level_complete`、`level_fail`
- `ai_question_asked`、`ai_filter_triggered`
- `project_created`、`project_saved`、`challenge_submitted`
- `ad_reward_clicked`、`parent_setting_changed`

**验收标准**：

- 事件只存本地，不上传儿童数据。
- 家长中心可展示基础统计。
- 后续可平滑替换为真实 analytics SDK。

### V105-P2-03 AR / 沉浸式任务 Spike

**背景**：海外报告将 AR/VR 教育列为长期趋势，但这不应抢占 V1.0.5 主线。

**建议范围**：

- 仅做技术预研或视觉样机。
- 方向可以是「AR 网格寻路」或「相机识别积木卡片」。
- 不进入 V1.0.5 必交付范围。

## 六、不建议在 V1.0.5 做的事

- 不建议开放真实儿童社区、自由评论或陌生人私信。
- 不建议把广告奖励做成学习通关的必要条件。
- 不建议继续新增大量静态页面，优先补核心闭环。
- 不建议把真实 API Key 写入 `server.py` 常量或前端 HTML。
- 不建议在没有家长同意流程前收集儿童真实身份信息。

## 七、建议排期

| 阶段 | 时间 | 交付 |
|---|---|---|
| 第 1 阶段 | 1 至 2 天 | 补齐 Python 代理交付包、README_PROXY、健康检查、前端代理状态 |
| 第 2 阶段 | 2 至 4 天 | Few-Shot 示例库、AI 输出复检、AI 指令解释 |
| 第 3 阶段 | 3 至 5 天 | 学习路径、关卡配置扩展、家长概念进度 |
| 第 4 阶段 | 3 至 5 天 | 创意编辑器 MVP、挑战提交、作品集 |
| 第 5 阶段 | 1 至 3 天 | 多语言文案字典、README/PRD 同步、验收测试 |

## 八、版本成功指标

| 指标 | 目标 |
|---|---|
| AI 可运行性 | 无需改 HTML，启动代理后即可真实对话 |
| 安全拦截 | 越狱、隐私、成人敏感、跑题输入均有稳定安全回复 |
| 学习闭环 | 至少 20 个关卡配置，概念进度可在家长中心查看 |
| 创作闭环 | 模板创建后可编辑、保存、提交挑战 |
| 海外化基础 | README_CN / README_TW / README_EN 完成，核心 UI 文案具备抽离方案 |
| 商业化安全 | 广告可由家长关闭，学习奖励不依赖广告 |
