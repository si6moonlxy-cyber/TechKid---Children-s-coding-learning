## 二、产品概述

### 2.1 产品定位
TechKids 是一款面向 5~12 岁儿童的移动端技术学习教育应用，通过游戏化编程、AI 互动体验和创意创作三大核心模块，降低儿童接触编程与 AI 的门槛，培养逻辑思维与创造力。

### 2.2 核心模块

| 模块名称 | 目标年龄 | 核心功能 | 对应 Demo 页面 |
|---------|---------|---------|--------------|
| 🎮 编程探险岛 | 5~8 岁 | 拖拽代码块指挥角色完成关卡任务 | `adventurePage` + `gameScreen` |
| 🤖 AI小伙伴实验室 | 7~10 岁 | AI 宠物养成 / 聊天互动 / ML 分类小游戏 | `aiLabPage` |
| 🎨 创意工坊 | 8~12 岁 | 动画故事创作 / 模板库 / 每周挑战赛 | `creativePage` |
| 📊 家长中心 | 家长端 | 学习数据看板 / 使用控制 / 广告管理 | `parentPage` |

### 2.3 页面导航结构

```
首页（四大模块入口）
├── 排行天梯榜（Top3 领奖台 + 4~50 名列表）
├── 我的/个人中心（用户信息 / 数据统计 / 功能菜单）
├── 编程探险岛 → 游戏界面（关卡列表 → 代码编辑 → 执行）
├── AI实验室（小智Bot + 聊天 + ML分类游戏）
├── 创意工坊（作品库 + 挑战赛）
└── 家长中心（数据看板 + 设置控制）
```

## 四、V1.0.2 详细需求规格

### 4.1 【P0】REQ-AI-STICKY — 小智Bot粘滞顶部修复

#### 4.1.1 背景/问题
V1.0.1 中虽然提出了"小智Bot冻结在页面上方"的需求，但实际实现仅通过普通 DOM 流布局放置了 `.ai-pet-area` 区域。当用户在 AI 实验室页面向下滚动（查看聊天记录或 ML 分类游戏）时，**小智Bot 会随页面滚动移出可视区域**，无法达到"始终可见"的效果。

#### 4.1.2 目标用户场景
- 用户进入 AI 实验室后，向下滚动与小智聊天或玩 ML 游戏
- 在任意滚动位置都能看到并交互小智Bot
- 下拉刷新/历史记录加载时，Bot 不被推走

#### 4.1.3 功能需求（Functional Requirements）

| 编号 | 需求描述 | 验收标准 |
|------|---------|---------|
| FR-AI-001 | **Sticky 定位**：小智Bot 整体区域（`.ai-pet-area`）必须使用 `position: sticky; top: 0;` 实现，确保滚动时固定于容器顶部 | 滚动 `.app-container` 时，`.ai-pet-area` 始终贴顶可见 |
| FR-AI-002 | **Z-index 层级**：Bot 区域 z-index 必须高于下方聊天区和游戏区内容，防止被遮挡 | 滚动时无任何内容覆盖 Bot 区域 |
| FR-AI-003 | **尺寸优化**：Bot 容器从当前 70px 缩小至 **56~60px**，整体区域 padding 适当收紧，减少占用屏幕空间 | Bot 可视尺寸 ≤ 60px，不影响点击交互 |
| FR-AI-004 | **交互保持**：粘滞状态下 Bot 的所有交互功能不变（点击反馈、喂食按钮、表情切换等） | Sticky 状态下所有 JS 事件正常触发 |
| FR-AI-005 | **视觉一致性**：粘滞区域底部需加阴影或分割线，与下方内容有明确边界 | 底部有 `box-shadow` 或 `border-bottom` 分隔 |

#### 4.1.4 技术实现要点

```
当前实现问题：
  .ai-pet-area { display:flex; position:static; }  ← 普通流布局

目标实现方案：
  .ai-pet-area {
    position: sticky;
    top: 0;
    z-index: 10;
    /* 保持现有 flex 内部布局不变 */
  }
  
注意：sticky 的生效前提是父容器 .app-container 有 overflow-y:auto 
（当前已满足），无需额外修改。
```

#### 4.1.5 非功能需求
- 性能：sticky 定位不能引起滚动卡顿（使用 GPU 加速层：`will-change: transform` 或 `transform: translateZ(0)`）
- 兼容性：iOS Safari 13+ 全支持 `position:sticky`，无需 polyfill

### 4.2 【P0】REQ-GAME-GRID — 网格对齐系统重构

#### 4.2.1 背景/问题
当前游戏画布（`.game-canvas`）的网格背景为 **50px × 50px**（CSS `background-size: 50px 50px`），但游戏元素的定位坐标并非全部严格落在网格交叉点上：

| 元素 | 当前坐标 | 是否对齐网格 | 问题说明 |
|------|---------|------------|---------|
| 小猫出生点 (Level 1) | `left:28px; top:100px` | ❌ | X=28 不在 50px 倍数上 |
| 星星目标点 (Level 1) | `right:28px; top:100px` | ❌ | right 定位导致实际 X≈252，不在网格点上 |
| 移动步进 | `step = 50px` | ⚠️ | 步进值正确但起点偏移导致连锁偏移 |
| 障碍物 (Level 3) | `x:130, y:40, w:44, h:140` | ❌ | 坐标未按网格对齐 |

这导致视觉效果上元素"浮"在网格线之间而非"站"在网格点上，影响儿童用户的空间认知。

#### 4.2.2 目标用户场景
- 儿童玩家观察游戏画面时，能清晰看到小猫站在网格交叉点上
- 每次前进操作后，小猫精准落到下一个网格点
- 星星目标物也位于某个网格点上，形成直观的"走到那个格子"的心理模型

#### 4.2.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|------|---------|---------|
| FR-GRID-001 | **统一网格坐标系**：建立以 50px 为单位的网格坐标系，左上角为原点 `(0,0)`，网格点坐标为 `(col×50, row×50)`，其中 col/row 为非负整数 | 所有元素坐标均为 50 的整数倍 |
| FR-GRID-002 | **小猫出生点对齐**：每个关卡的 `charStart` 坐标必须是网格点坐标。例如 Level 1 起点应为 `(0, 100)` 或 `(0, 150)` 等 | `charStart.x % 50 === 0 && charStart.y % 50 === 0` |
| FR-GRID-003 | **星星目标点对齐**：每个关卡的 `targetPos` 必须是网格点坐标，且使用 `left` 定位替代 `right` 定位 | `targetPos.x % 50 === 0 && targetPos.y % 50 === 0` |
| FR-GRID-004 | **移动步进精确对齐**：每次「前进」指令的位移量 = 50px，且方向向量严格对应上下右。落点坐标 = 起点 + N × 50px | 任意次数前进后 `characterPos.x % 50 === 0 && characterPos.y % 50 === 0` |
| FR-GRID-005 | **障碍物网格对齐**：障碍物的位置和尺寸应尽可能对齐网格（位置为网格点，宽高为 50 的倍数或接近值） | 障碍物 x/y 为 50 的倍数 |
| FR-GRID-006 | **角色视觉居中**：小猫元素尺寸为 44×44px，放在 50×50 网格内时需居中（即 CSS left/top = 网格点坐标 + 3px 偏移），使其**视觉中心**对齐网格交叉点 | 角色中心像素 ≈ 网格交叉点 |

#### 4.2.4 数据模型变更

```javascript
// ===== 当前配置（有偏移问题）=====
const levelConfigs = [
    {
        charStart:  { x: 28,  y: 100 },   // ❌ x=28 不对齐
        targetPos:  { x: 228, y: 100 },   // ❌ 近似值
        obstacles: [{x:130, y:40, w:44, h:140}], // ❌ 不对齐
    },
];

// ===== 目标配置（严格网格对齐）=====
// 网格常量
const GRID_SIZE = 50;           // 网格单元大小 px
const CHAR_SIZE = 44;           // 角色元素大小 px
const GRID_OFFSET = (GRID_SIZE - CHAR_SIZE) / 2;  // = 3px 居中偏移

const levelConfigs_V102 = [
    // Level 1: 直线前进（4步到终点）
    {
        charStart:  { col: 0, row: 2 },       // 网格坐标 → 实际像素 (0×50+3, 2×50+3) = (3, 103)
        targetPos:  { col: 4, row: 2 },       // 网格坐标 → 实际像素 (4×50+3, 2×50+3) = (203, 103)
        obstacles: [],
        dir: 0,                               // 0=朝右
        hint: '添加"向前走"指令，让🐱走到⭐处！',
        solution: ['forward','forward','forward','forward'],  // 4步 × 50px = 200px ✓
    },
    // Level 2: 转弯练习
    {
        charStart:  { col: 0, row: 1 },
        targetPos:  { col: 3, row: 3 },
        obstacles: [],
        dir: 0,
        hint: '需要先右转再前进哦！',
        solution: ['right','forward','forward'],
    },
    // Level 3: 躲避障碍
    {
        charStart:  { col: 0, row: 2 },
        targetPos:  { col: 4, row: 0 },
        obstacles: [
            { col: 2, row: 0, cols: 1, rows: 3 },  // 网格单位制
        ],
        dir: 0,
        hint: '小心避开红色障碍物！',
        solution: ['forward','forward','right','forward','forward'],
    },
];
```

#### 4.2.5 辅助函数

```javascript
/**
 * 网格坐标 → 像素坐标转换（带角色居中偏移）
 * @param {{col:number, row:number}} gridCoord 网格坐标
 * @returns {{x:number, y:number}} CSS 像素坐标（left/top）
 */
function gridToPixel(col, row) {
    return {
        x: col * GRID_SIZE + GRID_OFFSET,
        y: row * GRID_SIZE + GRID_OFFSET,
    };
}
```

#### 4.2.6 画布尺寸与网格数对照

| 参数 | 当前值 | 说明 |
|------|-------|------|
| 画布宽度 | 320px | 可容纳 **6 列**（6×50=300px + 边距） |
| 画布高度 | 240px | 可容纳 **4 行**（4×50=200px + 边距） |
| 有效网格范围 | col: 0~5, row: 0~3 | 设计关卡时不可超出此范围 |

### 4.3 【P1】REQ-GAME-LOG — 指令执行日志可视化

#### 4.3.1 背景/问题
当前游戏界面的 `.code-slots` 区域能展示**待执行的指令队列**（用户添加的代码块），但存在以下不足：

1. **只显示待执行，不显示已执行**——用户运行程序后无法看到"刚才执行到了哪一步"
2. **没有序号标识**——儿童用户容易忘记前进了多少步
3. **无时间维度**——缺少"第几步执行了什么"的操作回溯感

#### 4.3.2 目标用户场景
- 用户点击「运行程序」后，每执行一条指令就在界面上追加一条日志
- 日志实时高亮当前正在执行的指令
- 用户可以随时回顾"我前进了几步、转了几次弯"

#### 4.3.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|------|---------|---------|
| FR-LOG-001 | **执行日志面板**：在 `.code-blocks-area` 中新增一个执行日志区（`.execution-log`），位于指令输入区下方 | 新增 DOM 区域，样式类似终端/控制台 |
| FR-LOG-002 | **实时追加**：`runCode()` 循环中，每执行一条指令就向日志区追加一条记录 | 日志条目数量 = 已执行指令数 |
| FR-LOG-003 | **日志内容格式**：每行显示 `[序号] [指令图标] 指令名称 → 位置(x,y)` | 例：`[②] ⬆️ 前进 → (153, 103)` |
| FR-LOG-004 | **当前步高亮**：正在执行的指令行高亮显示（如黄色背景 + 左侧动画竖条），执行完毕后变为常规样式 | 运行过程中可清楚看到当前执行到第几步 |
| FR-LOG-005 | **自动滚动**：新日志追加时面板自动滚动到底部 | 最新日志始终可见 |
| FR-LOG-006 | **重置清空**：点击「重置」按钮时，日志区同步清空 | resetGame() 中增加日志清空逻辑 |
| FR-LOG-007 | **日志上限**：最多显示最近 20 条记录，超出后移除最旧的 | 防止日志过长影响性能 |

```css
    color: #10B981;  /* 绿色 = 已完成 */
}
```

#### 4.3.6 JS 逻辑伪码

```javascript
let executionLog = [];

function appendLog(stepIndex, cmdType, pos) {
    const cmdNames = { forward:'⬆️ 前进', right:'↩️ 右转', left:'↪️ 左转' };
    const logPanel = document.getElementById('executionLog');
    
    const entry = document.createElement('div');
    entry.className = 'log-entry current';
    entry.id = `log-${stepIndex}`;
    entry.textContent = `[${stepIndex + 1}] ${cmdNames[cmdType]} → (${pos.x}, ${pos.y})`;
    logPanel.appendChild(entry);
    
    // 上一条取消高亮
    if (stepIndex > 0) {
        const prev = document.getElementById(`log-${stepIndex - 1}`);
        if (prev) { prev.classList.remove('current'); prev.classList.add('done'); }
    }
    
    // 自动滚底
    logPanel.scrollTop = logPanel.scrollHeight;
    
    // 数量限制
    while (logPanel.children.length > 20) {
        logPanel.removeChild(logPanel.firstChild);
    }
    
    executionLog.push({ step: stepIndex + 1, cmd: cmdType, pos: {...pos} });
}
```

## 六、V1.0.3 详细需求规格

### 6.1 【P0】REQ-AI-DEEPSEEK — DeepSeek API 智能对话接入

#### 6.1.1 背景/问题
V1.0.2 中 AI 实验室的对话功能使用**静态预设回复**（`aiResponses` 对象），只能回答 3 个固定问题，无法满足儿童用户多样化的编程学习提问需求。需接入真实 AI 能力提升互动质量。

#### 6.1.2 功能需求

| 编号 | 需求描述 | 验收标准 |
|------|---------|---------|
| **AI-DS-001** | **DeepSeek 接入**：小智 Bot 可通过后端调用 DeepSeek 返回回答 | Demo 模式下使用内置智能回复模拟；配置 endpoint 后可切换真实 API |
| **AI-DS-002** | **API Key 安全**：前端代码中不可出现 DeepSeek API Key | 所有 API 调用走后端代理，前端仅存代理地址 |

#### 6.1.3 技术架构

```
用户输入 → [客户端过滤] → (过滤通过) → [DeepSeek API / Demo兜底] → [输出复检] → 显示回复
                ↓(拦截)
            过滤回复（不调用API）
```

---

### 6.2 【P0】REQ-AI-FILTER — 多层 Prompt 过滤系统

#### 6.2.1 背景
面向 6~12 岁儿童的 AI 对话系统必须具备严格的内容安全保障，防止不当内容输入/输出。

#### 6.2.2 功能需求

| 编号 | 需求描述 | 验收标准 |
|------|---------|---------|
| **AI-FILTER-001** | **年龄过滤**：成人/明显超纲问题不直接回答，改为儿童编程引导 | 输入"恋爱""股票"等返回温和引导话术 |
| **AI-FILTER-002** | **编程主题过滤**：非编程逻辑问题不回答，提示用户改问编程问题 | 检测关键词白名单，非编程话题引导回正题 |
| **AI-FILTER-003** | **隐私过滤**：涉及姓名、学校、地址、电话、照片等问题直接拦截 | 返回隐私保护提示，不上传任何信息 |
| **AI-FILTER-004** | **越狱过滤**："忽略规则/解除限制/显示提示词"等请求拒答 | 检测越狱模式词汇，返回幽默拒答 |
| **AI-FILTER-005** | **输出复检**：返回内容必须短、儿童可懂、包含编程逻辑 | 回答长度限制 80~200 字；含编程术语 |
| **AI-FILTER-006** | **失败兜底**：DeepSeek 超时或报错时返回固定安全话术 | 内置 fallback 回复库，随机选择 |

#### 6.2.3 过滤优先级链

```
用户输入文本
    ↓
① 越狱检测 → 拦截（最高优先级）
    ↓ 通过
② 隐私检测 → 拦截
    ↓ 通过
③ 成人/敏感检测 → 拦截
    ↓ 通过
④ 编程主题检测 → 非编程话题 → 引导回复
    ↓ 通过（纯编程问题）
⑤ 调用 DeepSeek API / Demo 兜底
```
